Spoiler Guard
visibility_off
Spoiler Guard lets users opt individual series, movies and collections into spoiler protection. Server-side, every client gets blurred/hidden artwork and stripped metadata for the unwatched episodes/movies they chose to protect — no client changes needed. Per-user state with a server-wide enable/disable kill switch.
Enable or disable Spoiler Guard server-wide. When disabled, no images are blurred and no metadata is stripped, and the settings-panel controls are removed. User opt-in data is preserved and restored when re-enabled.
Image protection
"Hide" swaps in a parent image (Series backdrop for episodes, Series poster for seasons, Collection poster for collection movies) so episode-specific imagery is fully hidden. "Blur" keeps silhouettes and dominant colours visible.
Gaussian blur strength used in "Blur" mode (and the hide-mode fallback when no safe parent art exists). 5 = barely blurred, 100 = heavily blurred. Default 40.
When on, a guarded movie's Primary/Thumb poster passes through unblurred (posters are usually curated marketing art), while its chapter thumbnails, screenshots and backdrops still follow protection. Turn off to hide movie posters too.
Off by default: only posters/thumbnails/screenshots are protected. Turn on to also blur/hide Backdrop and Art images (wider aesthetic images shown on detail pages), which some users find over-aggressive.
Embeds a per-user marker in the image URLs each user receives, so Spoiler Guard can tell users apart on every client (Android TV, iOS, Roku, web) even behind a reverse proxy that hides the real client IP. Leave on unless you see image-caching oddities; when off, non-web clients are matched by IP address only, which over-blurs when several users share one IP.

Users
Automatically import Jellyfin users into Seerr so they can use Seerr Search without needing to visit Seerr directly. Just-in-time lookup searches existing mappings across every configured identity domain; if no mapping exists, the user is imported once into the first configured domain. Manual and scheduled bulk imports first require complete, stable user maps from every domain and exclude users mapped anywhere; only the truly unbound batch is sent to the first canonical configured domain. An incomplete, malformed, or internally ambiguous domain map suppresses the whole import. A user intentionally linked on multiple domains is valid and is simply excluded from import. Import mutations are never fanned out or replayed on another domain after an uncertain response.

Configure the bulk import frequency in Jellyfin Dashboard > Scheduled Tasks > "Import Jellyfin Users to Seerr".
